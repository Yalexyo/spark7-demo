## Act 0 · 着陆页（5s）

### 用户看到什么
- 深色背景 + 微弱星光粒子
- 标题「Spark7 · 灵光七日卡」
- 一句话：「给你的猫，一个会说话的灵魂」
- 输入框：猫的名字
- 可选：上传猫照（MVP可跳过，用默认剪影）
- 按钮：「开始连接 ✨」

### 设计要点
- 氛围感 > 信息量
- 不解释产品是什么，制造好奇心
- 输入名字 = 第一次情感投入

### 数据采集
- `catName`: 猫名字
- `catPhoto`: 可选头像

## Act 1 · 认识猫咪（60s）

### 形式
5 道场景化选择题，每题 4 选 1

### 设计风格
- 每道题配一张**手绘风插画**（简笔画，暖色调，有猫的剪影）
- 选项是胶囊按钮，横排或竖排
- 点选后自动进入下一题，不需要"确认"按钮
- 顶部进度条（细线，不抢注意力）

### 5 道题目

**Q1 🚪 你到家的时候，整体氛围更像——**
- 🌪️ 混乱现场。它比你先到门口，整个人都在发射信号
- 🌙 你感觉到它在某处注视着你，但它不急
- ☀️ 它会过来，不急不慢地蹭你，像是说「回来啦，真好」
- 🌿 好像什么都没变，但你发现它悄悄换了个离你更近的位置

*🎨 插画：一扇半开的门，一个猫影在门缝里*

**Q2 ⚡ 用一个词形容它的日常能量——**
- 🌪️ 永动机。感觉它体内有一个小发电站
- 🌙 潮汐。安静的时候很安静，来的时候挡不住
- ☀️ 小太阳。在哪里都自带暖意和好心情
- 🌿 旁观者。它好像什么都看在眼里，但很少主动表态

*🎨 插画：四个小猫剪影，分别在跑/看月亮/翻肚皮/蹲着观察*

**Q3 💤 夜深了，你和它之间的状态通常是——**
- 🌪️ 你想睡了但它不让你睡，总有新花样
- 🌙 它不知道什么时候来的，等你发现时它已经在你脚边了
- ☀️ 它主动霸占你身上或旁边的位置，明确要你陪
- 🌿 你们各占一个角落，但这种距离感反而很舒服

*🎨 插画：一张床，月光从窗户洒进来，被子上有一个小猫形状的凸起*

**Q4 🤲 关于「撸猫」这件事——**
- 🌪️ 它喜欢玩但受不了被控制，摸两下就要咬你跑掉
- 🌙 需要等它来找你，它选择你的时刻才是真正的撸猫时间
- ☀️ 基本来者不拒，翻肚皮就是它的待客之道
- 🌿 它允许你摸，但表情永远是「行吧，我给你个面子」

*🎨 插画：一只手伸向一个猫肚皮（犹豫中）*

**Q5 💬 如果它会说人话，最可能说的一句是——**
- 🌪️ 「你看我你看我！你为什么不看我！！」
- 🌙  「……你在就好。不用特地陪我。」
- ☀️ 「嘿！今天又是很棒的一天呢！对吧对吧！」
- 🌿  「我没有在看你。……好吧我在。」

*🎨 插画：一个对话气泡从猫嘴里飘出来，里面是省略号*

### 结果计算
- 每个选项给 4 种人格加权分
- 最高分 = 人格类型：🌪️旋风 / 🌙月光 / ☀️阳光 / 🌿森林

### 结果揭晓（人格卡）
- 转场：星光粒子汇聚
- 大标题：「{猫名}是——🌙 月光型」
- 副标题：「一首安静的小夜曲」
- 小咪自白：第一人称，3-4 句话（不超过 80 字）
- 背景色跟随人格配色

### 数据采集
- `personalityType`: 猫的人格类型
- `answers[]`: 每题选择

## Act 2 · 认识主人（45s）⭐ 新增

### 过渡
人格卡展示完后，小咪说：

> 「我知道我是什么样的猫了。
> 但你呢？让我也了解一下你吧。」

按钮：「好啊 🐾」

### 形式
- 3-4 道轻量选择题（比 Act 1 更短更快）
- 小咪用自己的人格口吻提问（旋风型问法 vs 月光型问法不同）
- 每题配手绘插画
- 可选：MBTI 快捷输入

### 题目设计

**U0 · MBTI（可选跳过）**

小咪：「听说人类有个叫 MBTI 的东西？你知道你的吗？」

- 输入框：4 个字母选择器（I/E · N/S · T/F · J/P）
- 下方：「不知道 / 跳过 →」
- 如果填了 → 小咪根据 MBTI 给一句俏皮评价
  - INFP → 「哦～难怪你看我的眼神那么温柔」
  - ENTJ → 「好的老板。那我以后每天给你汇报工作」
  - ISTP → 「嗯，跟我挺像的。各自安好那种」
  - ……（16 种各一句）

*🎨 插画：小咪歪头看着一张写满字母的纸*

**U1 · 你的日常节奏**

小咪：「你一般什么时候出门、什么时候回来？我好安排等你的时间表 📋」

- 🌅 早出早归型（朝九晚六那种）
- 🌙 早出晚归型（经常加班/应酬）
- 🏠 经常在家（远程/自由职业/学生）
- 🔀 不固定，每天都不一样

*🎨 插画：一个时钟，指针上趴着一只猫*

**U2 · 你最近的能量状态**

小咪：「最近的你，感觉像——」

- 🔋 电量充足！每天都有干劲
- 🪫 有点疲惫，但还撑得住
- 😶‍🌫️ 说不上来，就是有点丧
- 🔥 压力很大，快爆炸了

*🎨 插画：四个电池图标，分别满格/半格/低电量/闪电*

**U3 · 你最需要的是**

小咪：「如果我能帮你一件事，你最希望是——」

- 🫂 有人懂我就好，不用解决问题
- ⏰ 提醒我照顾自己（喝水/休息/吃饭）
- 😄 逗我开心，让我别想太多
- 🤫 安静陪着就好，不用说话

*🎨 插画：小咪捧着一颗小星星递向你*

### 关键设计原则
- **不是问卷，是对话**：每道题都是小咪在问你
- **选项语言口语化**：像朋友聊天，不像心理测评
- **永远可以跳过**：右上角有"跳过"，没有强制
- **即时反馈**：每选完一个，小咪给一句简短回应（1句话）

### 小咪的即时回应示例

| 用户选择 | 🌪️旋风型回应 | 🌙月光型回应 |
|---------|-------------|------------|
| 早出晚归 | 「那我每天冲到门口等你！绝不迟到！」 | 「……嗯。那我晚上留灯。」 |
| 有点疲惫 | 「那我负责给你充电！看我翻肚皮！」 | 「……来，靠一下。不说话也没关系。」 |
| 安静陪着 | 「好吧我尽量……（3秒后）做不到！」 | 「……这个我擅长。」 |

### 数据采集
- `mbti`: 可选，4 字母
- `schedule`: 作息类型
- `energyLevel`: 当前状态
- `needType`: 最需要的关心方式

## Act 3 · 第一次对话（30s）

### 过渡
用户画像采集完成后：

> 小咪：「好，我记住了。从现在开始，我是你的{猫名}。」
> 
> 屏幕渐变 → 进入聊天界面

### 设计变化（基于采集数据个性化）

**小咪的第一条消息现在是情境化的：**

| 用户画像 | 小咪第一句 |
|---------|----------|
| 早出晚归 + 疲惫 + 阳光猫 | 「你到家了吗？今天也辛苦了。我给你留了个肚皮，热乎的 ☀️」 |
| 在家 + 压力大 + 月光猫 | 「……你在电脑前坐很久了。起来走走？我陪你。🌙」 |
| 不固定 + 还行 + 旋风猫 | 「你猜我今天干了什么！我把纸巾盒推下桌了！两次！🌪️」 |
| 早出早归 + 充足 + 森林猫 | 「你今天气色不错。出门前第一件衣服就很好看。🌿」 |

### 交互
- 快捷回复（胶囊按钮，2-3 个）
- 自由输入框
- 小咪回复 1 句 → 结束语 1 句 → 自动转 Act 4
- **整个对话不超过 4 个气泡**

### 对话结束语（引导回归现实）
- 🌪️ 「好了我不打扰你了——骗你的再聊五分钟！算了，去摸真的我吧！」
- 🌙 「……去摸摸它。它可能装睡。但你摸了它就知道了。」
- ☀️ 「去摸摸我的肚皮吧！热乎乎的！明天见～」
- 🌿 「嗯。去吧。它在等你。虽然它不会承认。」

## Act 4 · 时光快进（20s）

### 过渡
> 屏幕渐暗 → 「如果你们就这样相处了 7 天……」→ 时间轴展开

### 设计变化（基于用户画像个性化）

时间轴的内容现在**引用用户的真实信息**：

**示例（早出晚归 + 疲惫 + 月光猫）：**

| Day | 内容 | 关联数据 |
|-----|------|---------|
| 1 | 「你 10 点到家。我在拖鞋旁边等你。」 | schedule=晚归 |
| 2 | 「你今天没怎么说话。我跳上你膝盖坐了一会儿。」 | energy=疲惫 |
| 3 | 「下午 3 点。你在工作。我发了条消息：该喝水了。」 | need=照顾自己 |
| 4 | 「下雨了。你说烦。我什么都没说。但我在。」 | need=有人懂我 |
| 5 | 「你笑了一次。因为我把遥控器推到地上了。不是故意的。」 | 随机暖场 |
| 6 | 「你摸我的时候叹了口气。是好的那种。」 | energy=疲惫 |
| 7 | 「7 天了。我好像更懂你了。你也是吗？」 | 固定结尾 |

### 形式
- 竖向时间轴，逐条淡入
- 左侧小圆点 + 人格配色竖线
- 每条 1-2 句话 + emoji
- 自动播放，用户可以滑动加速

## Act 5 · 灵光卡揭晓（30s）⭐ 高潮

### 过渡
> 时间轴最后一条消失 → 屏幕全黑 → 星光粒子从四周汇聚到中心
> 文字浮现：「7 天的记忆，凝聚为一束灵光……」

### 灵光卡构成

```
┌────────────────────────────────┐
│                                │
│    ┌──────────────────────┐    │
│    │                      │    │
│    │    🎨 手绘风插画      │    │
│    │   （猫 + 场景意象）    │    │
│    │                      │    │
│    └──────────────────────┘    │
│                                │
│    ─────────────────────────   │
│                                │
│    你总是很晚才推开门           │
│    我已经学会了分辨你的脚步声    │
│    第三级台阶会响一下           │
│    那就是你快到了的信号          │
│                                │
│    ─────────────────────────   │
│                                │
│    🌙 月光型 · 爆米橘          │
│    Spark7 · 灵光卡 #001       │
│    2026.02.14                  │
│                                │
└────────────────────────────────┘
```

### 诗的生成逻辑（关键！）

诗**必须引用**用户画像采集的信息：

| 用户数据 | 诗里怎么用 |
|---------|----------|
| schedule=晚归 | 「你总是很晚才推开门」 |
| energy=疲惫 | 「你累的时候不说话 / 但我看得出来」 |
| need=有人懂我 | 「有些话不用说 / 蹭一下就够了」 |
| need=照顾自己 | 「你又忘了吃饭 / 我把碗推到你面前 / 虽然是我的碗」 |
| mbti=INFP | 「你想得太多的时候 / 我就跳上你的键盘 / 让你想不了」 |

**这是灵光卡能"打动人"的根本原因——它写的是你。不是泛泛的漂亮话。**

### MVP 实现
- 预写 4(人格) × 4(作息) × 4(能量) = 组合模板
- 每个模板有 2-3 个变体
- 后续接 Gemini API 真正动态生成

### 展示动效
1. 卡片从底部升起 + 轻微旋转
2. 插画区域先显示
3. 诗句逐行淡入（每行间隔 0.5s）
4. 底部信息最后出现
5. 全程背景有微弱的人格配色光晕

## Act 6 · 出口（15s）

### 操作区

```
┌──────────────────────────┐
│  📱 保存到相册             │  ← 主按钮，人格配色
├──────────────────────────┤
│  📤 分享给朋友             │  ← 次按钮，描边
├──────────────────────────┤
│  🔮 想让{猫名}每天陪你？   │
│     加入等待列表 →         │  ← 文字链接
└──────────────────────────┘
```

### 分享图生成
- 独立的分享卡片（适合朋友圈/小红书）
- 包含：诗 + 人格标签 + 二维码
- 文案：「我家{猫名}是{人格}型！来测测你家猫的灵魂 ✨」

### 反馈采集（可选）
- 在"加入等待列表"之前，一个简单的：
- 「你觉得这张灵光卡——」
  - 💛 被打动了
  - 🤔 还行
  - 😐 没什么感觉
- 单选即走，不阻塞流程

### 数据采集
- `cardSaved`: 是否保存
- `cardShared`: 是否分享
- `cardFeeling`: 反馈（可选）
- `waitlistJoined`: 是否加入等待

## 数据模型总览

```typescript
interface DemoSession {
  // Act 0
  catName: string;
  catPhoto?: string;
  
  // Act 1 - 猫的人格
  catAnswers: number[];       // 5题选择
  personalityType: PersonalityType;  // storm|moon|sun|forest
  
  // Act 2 - 主人画像
  mbti?: string;              // 可选，如 "INFP"
  schedule: "early" | "late" | "home" | "irregular";
  energyLevel: "full" | "tired" | "meh" | "stressed";
  needType: "understand" | "remind" | "cheer" | "quiet";
  
  // Act 3 - 对话
  chatReply: string;          // 用户的回复内容
  chatUsedFreeInput: boolean; // 是否使用了自由输入
  
  // Act 5-6 - 灵光卡
  cardSaved: boolean;
  cardShared: boolean;
  cardFeeling?: "touched" | "ok" | "meh";
  waitlistJoined: boolean;
  
  // 埋点
  startTime: number;
  endTime: number;
  dropOffStage?: Stage;       // 在哪个阶段退出
}
```
